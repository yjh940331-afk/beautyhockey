# 🏒 beautyhockey

> **아마추어 중에 가장 잘생기고, 가장 하키 잘하는 남자가 되는 프로젝트.**
> 목표 기간: 30년. 60대에도 빙판 위에 있는 것.

세 목표(하키 퍼포먼스 / 체지방 10% / 외모·피부)를 **운동을 더하는 게 아니라, 충돌하지 않게 룰로 관리**하는 저장소.
플랜은 도구고, **측정 → 조정 → 측정**의 사이클이 진짜 시스템이다.

## 🔒 5대 원칙 (안 지켜지면 어떤 스케줄도 무의미)

1. **무릎이 트레이닝의 상한선.** 통증 4일 이상 = 즉시 1주 휴식 (협상 X)
2. **수면 = 회복의 단일 최강 변수.** 80% 미만 주 2연속 → 강제 디로딩
3. **하체 헤비는 다음 게임 48시간+ 전(수 점심)에만, 플라이오는 저볼륨 폭발성 한정 24시간 전(목 저녁)에만.** 그 외 요일 하체 자극 추가 금지
4. **카페인은 13:00까지** (금 12:00)
5. **하루 3세션 = 목요일만**

우선순위 고정: **① 무릎/건강 → ② 하키 퍼포먼스 → ③ 컷팅/외모** → 충돌 판정은 [docs/06_decision_rules.md](docs/06_decision_rules.md)

## 🔁 운영 방법

```bash
# 매일 (상세 기록이 필요한 날만; 평소엔 주간 파일 표에 한 줄)
cp templates/daily_log.md logs/2026-07-09.md

# 매주 (일요일 게임 후 or 월요일)
cp templates/weekly_review.md logs/2026-W29.md
# 작성 → 신호등 판정 → STATE.md 갱신
git add . && git commit -m "log: 2026-W29 weekly review" && git push
```

**최소 로깅 룰:** 다 못 채워도 **아침 3숫자(RHR / 수면 / 무릎)만은 매일.** 이 3개가 모든 판정의 입력값이고, 나머지 칸은 비어도 시스템이 돌아간다. 완벽주의로 로깅 자체를 포기하는 게 최악.
